# MedVault

A local-first, privacy-first personal health records manager for Android.

MedVault lets patients consolidate all their medical records — prescriptions, lab reports, X-rays — on their own phone, with zero data sent to any cloud or server.

## Setup

### Prerequisites
- Android Studio Ladybug (2024.2) or later
- JDK 17
- Android SDK 34

### Getting started
1. Clone this repository
2. Open in Android Studio
3. Sync Gradle (Studio handles the Gradle wrapper automatically)
4. Run on an emulator or device (min SDK 26 / Android 8.0)

> **Note**: This project uses Expo Dev Client / development builds for native modules. It will **not** work in Expo Go.

## Architecture

**MVVM** with ViewModels, StateFlow, and the Repository pattern.

```
UI (Compose) → ViewModel → Repository → Room Database / File System
```

- **Screens** observe `StateFlow` from ViewModels
- **ViewModels** call repository methods
- **Repositories** run on `Dispatchers.IO` and coordinate DAO access
- **DAOs** are auto-generated by Room

## Room Schema

| Table | Description |
|-------|-------------|
| `visits` | Core visit records (date, doctor, diagnosis, etc.) |
| `prescriptions` | One-to-one with visits. Medicines stored as JSON. |
| `attachments` | One-to-many with visits. File paths to images/PDFs. |
| `contacts` | Doctor autocomplete cache |
| `visit_fts` | FTS4 virtual table for full-text search |

Foreign keys with `CASCADE` delete ensure referential integrity.

## File Layout

```
context.filesDir/
  medvault/
    prescriptions/
      {visitId}.jpg          ← prescription photos
    media/
      {visitId}/
        {mediaId}.jpg        ← X-rays, before/after photos
        {mediaId}.pdf        ← lab reports
```

All paths stored in Room as **relative paths**. Resolved to absolute by prepending `context.filesDir.absolutePath` at read time.

## Phase 2 — AI Features (Roadmap)

All AI integration points are marked with `// TODO: PHASE 2 — AI` in the codebase.

To enable AI summaries:
1. Implement `AiSummaryRepository` with Retrofit/OkHttp
2. Add API key configuration in Settings (with privacy warning)
3. Add AI summary card to Visit Detail screen
4. Add "Generate AI Summary" button to prescription step

The `AiSummaryRepository` is the **only** class that needs to change to swap AI providers (Gemini → OpenAI → on-device model).

## Data Backup & Restore

### Export
Settings → Data → Export all data as ZIP
- Creates a ZIP containing `medvault.db` + all media files
- Share via system share sheet

### Import
Settings → Data → Import data from ZIP
- Replaces current database and media files
- Confirmation dialog before proceeding

### Manual backup
Copy from `context.filesDir/medvault/` and the database file `medvault.db`.

## Adding a Room Migration

1. Update the entity class with the new field
2. Increment the database version in `MedVaultDatabase.kt`
3. Add a migration in `data/db/migrations/Migrations.kt`
4. Register the migration in `di/DatabaseModule.kt`

Never use `fallbackToDestructiveMigration()`.
